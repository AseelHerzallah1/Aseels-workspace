# Ask Aseel's Bot — Backend (FastAPI + RAG)

A small Python backend that powers the personal "Ask Aseel's Bot". It uses
**retrieval-augmented generation (RAG)**: your profile lives in markdown files,
gets embedded into vectors, and the most relevant pieces are retrieved and fed
to a Groq LLM so answers stay grounded in verified info.

## How it works

```
content/*.md  →  chunk  →  embed (local model)  →  data/knowledge.json
                                                          │
question  →  embed  →  cosine similarity  →  top chunks ──┘
   │
   └─→  injected into Groq prompt  →  grounded streamed answer
```

## Project layout

```
backend/
├── app/
│   ├── main.py        # FastAPI app + /chat endpoint
│   ├── rag.py         # chunking, embeddings, cosine search
│   ├── chat.py        # grounded prompt + Groq streaming
│   └── config.py      # env-based settings
├── content/           # your knowledge base (edit these!)
├── scripts/build_index.py
├── data/knowledge.json  # generated by build_index.py
└── requirements.txt
```

## Run locally

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate          # Windows
# source .venv/bin/activate     # macOS/Linux

pip install -r requirements.txt
copy .env.example .env          # then add your GROQ_API_KEY

python scripts/build_index.py   # build the vector index
uvicorn app.main:app --reload --port 8000
```

Check it: open http://localhost:8000/health

## Updating your info

1. Edit any file in `content/`.
2. Re-run `python scripts/build_index.py`.
3. Restart the server (or it picks up on next start).

## Endpoints

- `GET /health` — status + number of indexed chunks
- `POST /chat` — body: `{ "messages": [{ "role": "user", "content": "..." }], "mode": "portfolio" }`; streams plain text. `mode` is one of `portfolio`, `recruiter`, `job_match`.
